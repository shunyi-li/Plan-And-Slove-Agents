# Plan-And-Solve Agents

## 项目概述

Plan-And-Solve Agents 是一个基于大语言模型的智能任务处理系统，它能够将复杂问题分解为一系列可执行的步骤，然后逐步执行这些步骤以解决问题。该系统采用了规划-执行分离的架构，通过 Planner 生成详细的行动计划，再由 Executor 按照计划逐步执行，最终得到问题的解决方案。

## Plan-and-Solve 的工作原理
Plan-and-Solve Prompting 由 Lei Wang 在2023年提出[2]。其核心动机是为了解决思维链在处理多步骤、复杂问题时容易“偏离轨道”的问题。

与 ReAct 将思考和行动融合在每一步不同，Plan-and-Solve 将整个流程解耦为两个核心阶段：

规划阶段 (Planning Phase)： 首先，智能体会接收用户的完整问题。它的第一个任务不是直接去解决问题或调用工具，而是将问题分解，并制定出一个清晰、分步骤的行动计划。这个计划本身就是一次大语言模型的调用产物。
执行阶段 (Solving Phase)： 在获得完整的计划后，智能体进入执行阶段。它会严格按照计划中的步骤，逐一执行。每一步的执行都可能是一次独立的 LLM 调用，或者是对上一步结果的加工处理，直到计划中的所有步骤都完成，最终得出答案。

## 功能特点

- **智能问题分解**：将复杂问题自动分解为多个简单步骤
- **分步执行**：按照逻辑顺序逐步执行计划中的每个步骤
- **流式响应**：实时展示大语言模型的思考过程
- **灵活配置**：支持自定义LLM模型和API参数
- **错误处理**：具备基本的错误处理和异常捕获能力

## 系统架构

系统由以下核心组件组成：

1. **PlanAndSolveAgent**：智能体核心，协调规划和执行过程
2. **Planner**：负责将问题分解为具体的行动计划
3. **Executor**：负责按照计划逐步执行并解决问题
4. **LLM**：大语言模型客户端，用于与LLM服务交互
5. **Config**：配置管理，处理环境变量和提示词模板

## 使用方法

### 基本使用

```python
from plan_and_solve_agent import PlanAndSolveAgent
from llm_service import LLM

# 初始化LLM客户端
llm_client = LLM()

# 创建智能体实例
agent = PlanAndSolveAgent(llm_client)

# 提出问题
question = "一个水果店周一卖出了15个苹果。周二卖出的苹果数量是周一的两倍。周三卖出的数量比周二少了5个。请问这三天总共卖出了多少个苹果？"

# 运行智能体
agent.run(question)
```

## 核心组件说明

### PlanAndSolveAgent

智能体的核心类，负责协调规划和执行过程。它接收一个LLM客户端实例，并使用它来创建Planner和Executor实例。

### Planner

负责将复杂问题分解为一系列可执行的步骤。它使用大语言模型生成一个详细的行动计划，然后解析并返回这个计划。

### Executor

负责按照计划逐步执行每个步骤。它使用大语言模型来解决每个步骤的问题，并跟踪执行历史，最终返回最终答案。

### LLM

大语言模型客户端，用于与LLM服务交互。它支持流式响应，可以实时展示模型的思考过程。

## 配置说明

项目使用环境变量进行配置，主要配置项包括：

- `LLM_API_KEY`：LLM服务的API密钥
- `LLM_BASE_URL`：LLM服务的基础URL
- `LLM_MODEL_ID`：使用的LLM模型ID
- `LLM_TIMEOUT`：LLM服务的超时时间（可选，默认60秒）

## 扩展与定制

### 自定义提示词模板

您可以在 `config.py` 文件中修改 `PLANNER_PROMPT_TEMPLATE` 和 `EXECUTOR_PROMPT_TEMPLATE` 来定制提示词模板，以适应不同的任务类型。

3. 集成更多工具和服务
4. 添加结果验证和反馈机制

## 应用场景

- **数学问题求解**：分解复杂数学问题并逐步解决
- **逻辑推理**：处理需要多步推理的逻辑问题
- **任务规划**：为复杂任务生成详细的执行计划
- **教育辅助**：帮助学生理解问题解决的步骤和过程
- **决策支持**：为决策过程提供结构化的分析步骤

## 注意事项

- 系统依赖于LLM服务的可用性和响应质量
- 复杂问题可能需要更详细的提示词模板和更多的执行步骤
- 系统目前仅支持文本输入和输出，不支持其他类型的输入（如图像、音频等）
